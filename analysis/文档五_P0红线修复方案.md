## G1. I2 / I3：IPC 鉴权方案 + 协议 CSP 收敛

### 1. 根因

- **I2（无鉴权）**：`ipc-handlers.ts:17` `ipcMain.handle('api:call', async (event, { url, options = {} }) => {` 入口**不做任何调用方身份/令牌校验**。`preload.ts:9` 通过 `sendSync('system:getApiToken')` 取得 `apiToken` 并暴露（`preload.ts:15`），但主进程从不比对——等价于「信任任意渲染内脚本」，可调用文件读取、沙盒执行、权限授予等全部能力。
- **I3（CSP 绕过）**：`main.ts:23-35` `protocol.registerSchemesAsPrivileged([{ scheme:'claw', privileges:{ bypassCSP:true, ... }}])` 使 `claw://` 页面不受 CSP 约束；一旦渲染层出现 XSS（见 I4），可无阻执行脚本/外联，与 I2 叠加放大危害。

### 2. 修复设计

#### 2.1 `api:call` 入口鉴权中间件（新增，落于 `ipc-handlers.ts`）

`main.ts:19` 已生成 `const apiToken = randomBytes(32).toString('hex')`，将其传入 `registerApiIpc` 闭包作为 `EXPECTED_TOKEN`。在 `api:call` 首行加：

```ts
// ipc-handlers.ts —— api:call 入口
ipcMain.handle('api:call', async (event, { url, options = {} }) => {
  assertOrigin(event);                       // ① origin 白名单
  const token = options.__token;            // ② 由 preload 注入
  if (token !== EXPECTED_TOKEN) {
    throw new Error('FORBIDDEN: invalid api token');
  }
  assertPermission(url, options.method);    // ③ RBAC 授权（见 G3）
  // ... 原有路由分发
});

function assertOrigin(event: any) {
  const url = (event.senderFrame?.url) ?? event.sender.getURL();
  if (!url.startsWith('claw://') && !url.startsWith('file://')) {
    throw new Error('FORBIDDEN: invalid origin ' + url);
  }
}
```

`preload.ts` 改造：`apiCall` 自动注入 token（**不在渲染层暴露明文 token 给业务代码**，仅用于内部鉴权）：

```ts
// preload.ts
apiCall: (url, options) =>
  ipcRenderer.invoke('api:call', { url, options: { ...options, __token: apiToken } }),
```

> 说明：token 仅在渲染进程内用于通过主进程校验，真正的纵深防御依赖 `contextIsolation`（已开启）+ origin 白名单 + RBAC。token 本身不对抗 XSS（XSS 可读到），但与 I3 CSP 收敛组合后，XSS 注入面被大幅压缩。

#### 2.2 CSP 收敛（`main.ts` + `index.html`）

- 移除 `main.ts:30` `bypassCSP: true`，保留 `secure/standard/supportFetchAPI/corsEnabled`：
  ```ts
  privileges: { secure: true, standard: true, supportFetchAPI: true, corsEnabled: true, stream: true }
  // 删除 bypassCSP
  ```
- 在 `dist/renderer/index.html` `<head>` 注入（随 provider 域名动态生成白名单）：
  ```html
  <meta http-equiv="Content-Security-Policy" content="
    default-src 'self' claw:;
    script-src 'self';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: blob:;
    connect-src 'self' http://127.0.0.1:* https://api.openai.com https://api.anthropic.com;
    object-src 'none'; base-uri 'self'; form-action 'none';">
  ```
- `main.ts:407-420` 的 `protocol.handle('claw', ...)` 增加**目录白名单**（关联 I7/待核实 T4）：解析后 `filePath` 必须位于渲染 `dist` 目录或 `userData` 允许子目录内，禁止 `..` 穿越。

### 3. 验收标准
- [ ] 渲染层外域 `iframe`/注入 `<script>` 调用 `api:call` 被 `assertOrigin` 拒绝。
- [ ] 缺少/错误 `__token` 的调用返回 `FORBIDDEN`。
- [ ] 移除 `bypassCSP` 后，标准 XSS payload 被 CSP 拦截（在 claw:// 页面注入 `<script>alert(1)</script>` 不执行）。
- [ ] `claw://` 协议读取越权路径（如 `claw://app/../../etc/passwd`）被拒。

### 4. 影响面
涉及 `main.ts`、`preload.ts`、`ipc-handlers.ts`；为**所有 IPC 通道的前置安全层**，与 G3 RBAC 网关强耦合。

### 5. 建议工期：**3 人日**（前端 1 + 架构/后端 2）。

## G2. S1 / S2：沙盒加固（命令解析分级 + 降级 env 裁剪）

### 1. 根因

- **S1（正则绕过）**：`sandbox.ts:7-42` 仅用 `HIGH_RISK_PATTERNS`/`MEDIUM_RISK_PATTERNS` 正则；`getRiskLevel`（`sandbox.ts:182-190`）对 `node -e "..."` / `python -c "..."` / `pwsh -c "..."` 等**外层载体不在名单** → 判 `'low'` → `execute`（`sandbox.ts:218-238`）**直接执行且无需确认**，内含破坏性 payload 可零确认运行。
- **S2（降级继承密钥）**：`_runCommand`（`sandbox.ts:279-413`）无 Docker 时 `spawn(shellName, shellArgs, { env: { ...process.env } })`（`sandbox.ts:365`）继承含 API Key/Token 的宿主环境，叠加 S1 绕过 = 高危。

### 2. 修复设计

#### 2.1 命令解析风险引擎（替换正则分级）

新增 `src/backend/risk-engine.ts`（或在 `sandbox.ts` 内），要点：

```ts
const INTERPRETER_CARRIERS = ['node','node.exe','python','python3','pwsh','powershell','cmd','sh','bash','ruby','perl','wscript','cscript'];
const FORBIDDEN_PATTERNS = [/\brm\s+-rf\b/i, /\bformat\s+[a-z]:/i, /\bshutdown\b/i, /\bREG\s+ADD\b/i, />\s*\/dev\/(sd|hd)/i, /\b(iwr|curl|wget)\b.*\$/i];
const HIGH_PATTERNS = [/\b(del|rmdir|rm|remove-item)\b/i, /\b(reg|regedit)\b/i, /\b(sc\s+delete|Stop-Service)\b/i, /\bInvoke-Expression\b/i, /\bSet-ExecutionPolicy\b/i];
const MEDIUM_PATTERNS = [/\b(copy|move|rename|mkdir|New-Item)\b/i, /\b(net\s+(user|share|use|localgroup))\b/i, /\b(pip|npm|yarn)\s+install\b/i];

export function assessRisk(command: string): 'forbidden'|'high'|'medium'|'low' {
  const { interpreter, script } = parseCommand(command);   // 提取首 token 与 -e/-c/-Command 之后的脚本体
  const body = script ?? command;
  if (FORBIDDEN_PATTERNS.some(p => p.test(body))) return 'forbidden';
  if (interpreter && INTERPRETER_CARRIERS.includes(interpreter)) return 'medium'; // 解释器载体一律确认
  if (HIGH_PATTERNS.some(p => p.test(body))) return 'high';
  if (MEDIUM_PATTERNS.some(p => p.test(body))) return 'medium';
  return 'low';
}
```

`execute()` 改为：

```ts
const risk = assessRisk(command);
if (risk === 'forbidden') return { blocked: true, message: '⛔ 命令命中禁止清单，已拒绝执行' };
if (risk === 'high' || risk === 'medium') return { needsConfirmation: true, riskLevel: risk, command };
// low 且已授权 → 直接执行
```

#### 2.2 无 Docker 降级 env 裁剪（S2）

`sandbox.ts:362-366` 改为构造**最小化 env**，裁剪敏感变量；高/中危命令在降级模式下强制确认并提示：

```ts
const SENSITIVE = /^(OPENAI_API_KEY|ANTHROPIC_API_KEY|AWS_|AZURE_|GCP_|OPENCLAW_|.*_TOKEN|.*_SECRET|.*API_KEY|.*PASSWORD)/i;
const safeEnv: Record<string,string> = {};
for (const [k, v] of Object.entries(process.env)) {
  if (!SENSITIVE.test(k)) safeEnv[k] = v as string;
}
// 进一步收窄 PATH 到最小可信集（仅系统必要目录）
const proc = spawn(shellName, shellArgs, { cwd, windowsHide: true, env: safeEnv });
```

- 新增配置 `sandbox.allowHostFallback`（默认 true），但 **high 风险命令在 fallback 模式下默认禁止自动执行**，仅 low/medium 并经用户确认；UI 明确提示「当前为宿主机降级模式，建议安装 Docker 获得强隔离」。

### 3. 验收标准
- [ ] `node -e "process.exit(1)"`、`python -c "import os;os.system('rm -rf /tmp/x')"` 触发 `needsConfirmation`，**不静默执行**。
- [ ] `rm -rf /`、`format c:`、`shutdown /s` 类命令被 `forbidden` 直接阻断。
- [ ] 宿主机降级执行时，子进程 `process.env` 不包含任何 `*_KEY`/`*_SECRET`/`*_TOKEN`/`OPENCLAW_*`（通过 `env` 打印验证）。
- [ ] 有 Docker 时仍走容器隔离（回归不退化，`--network none` 保留）。

### 4. 影响面

## G3. S3 / S4：RBAC 网关落地 + 自动化命令参数化

### 1. 根因

- **S3（RBAC 未生效）**：`permission-manager.ts` 已实现 `checkPermission(resource, action, role)`（`permission-manager.ts:109`），但 `ipc-handlers.ts` 仅在 `/permissions`（`ipc-handlers.ts:1501`）与 `/permissions/roles`（`1509`）使用；`api:call` 无网关；`DEFAULT_PERMISSIONS.currentRole='admin'`（`permission-manager.ts:56`），且无登录/身份概念。

### 2. 修复设计

#### 2.1 RBAC 作为 `api:call` 网关（S3）

- 将 `permission-manager.ts:56` 默认角色改为 `'user'`。
- 在 `ipc-handlers.ts` 增加路由→权限映射（与 G1 的 `assertPermission` 合并）：

```ts
const ROUTE_PERMISSIONS: Record<string, {resource:string; action:string}> = {
  '/sandbox/execute':           { resource:'sandbox',    action:'execute' },
  '/sandbox/permissions':       { resource:'sandbox',    action:'write' },
  '/knowledge/ingest-url':      { resource:'skill',      action:'write' },
  '/knowledge/sources':         { resource:'skill',      action:'write' },
  '/automation/screenshot':     { resource:'automation', action:'execute' },
  '/settings':                  { resource:'settings',   action:'write' },
  '/models':                    { resource:'model',      action:'write' }, // POST 写
  // GET 类 read 默认放行（或在映射中标 read）
};
function assertPermission(url: string, method: string) {
  const perm = matchRoute(ROUTE_PERMISSIONS, url, method);
  if (perm && !permissionManager.checkPermission(perm.resource, perm.action)) {
    throw new Error('FORBIDDEN: insufficient permission');
  }
}
```

- 形成「**origin+token 鉴权（G1）→ RBAC 授权（本项）**」两段式网关。

- 新增 `escapePowerShell*` 工具并**消除字符串拼接**：
  - `typeText`：用 Base64 编码文本，PowerShell 侧解码，**不拼入脚本字面量**：

#### 2.2 SSRF 防护 + 知识泵默认关（K2）

```ts
import dns from 'dns';
const PRIVATE = [/^10\./, /^172\.(1[6-9]|2\d|3[01])\./, /^192\.168\./, /^127\./, /^169\.254\./, /^::1$/, /^fc00:/, /^fe80:/];
export async function assertSafeUrl(url: string, opts: { allowlistDomains?: string[] }) {
  const u = new URL(url);
  if (u.protocol !== 'http:' && u.protocol !== 'https:') throw new Error('scheme blocked');
  if (opts.allowlistDomains && !opts.allowlistDomains.includes(u.hostname))
    throw new Error('domain not in allowlist');
  const ips = await dns.lookup(u.hostname, { all: true });
  for (const { address } of ips) if (PRIVATE.some(re => re.test(address))) throw new Error('private ip blocked: ' + address);
  // 重定向到内网：fetch 后校验最终 URL（若发生重定向）
}
```

- `DataCrawler.crawlUrl` 调用前 `await assertSafeUrl(url, { allowlistDomains: settings.crawlingAllowlist })`。
- `knowledge-pump.ts:37` 默认 `urls = []`（**默认关闭自动爬网**），仅当用户在 `settings.json` 显式配置 `crawlingEnabled:true` 且 `crawlingSources` 非空才推送；加频率配额（每源每日上限）+ 审计日志（记录外联 URL/时间/结果）。

## G5. C1：密钥管理方案（系统密钥库 + 每机派生）

### 1. 根因

`model-manager.ts:16` `const FALLBACK_SECRET = 'openclaw-default-api-key-safe-salt';`，在 `encryptText`/`decryptText` 中 `crypto.scryptSync(FALLBACK_SECRET, 'salt-claw', 32)`（`model-manager.ts:30,59`）。常量公开 → 任何人可离线解密所有 `fallback:` 前缀密文，加密形同明文混淆。

### 2. 修复设计

- 新增 `src/backend/secret-store.ts`，**运行时派生主密钥**，废除硬编码常量：

```ts
import { safeStorage } from 'electron';
import * as fs from 'fs'; import * as path from 'path'; import * as crypto from 'crypto';
const KEY_PATH = path.join(dataDir, '.keyring', 'app.key');

export function getMasterKey(): Buffer {
  if (safeStorage?.isEncryptionAvailable()) {
    const encPath = KEY_PATH + '.enc';
    if (!fs.existsSync(encPath)) {
      const k = crypto.randomBytes(32).toString('hex');
      fs.mkdirSync(path.dirname(encPath), { recursive: true });
      fs.writeFileSync(encPath, safeStorage.encryptString(k)); // 系统密钥库托管
    }
    return Buffer.from(safeStorage.decryptString(fs.readFileSync(encPath)), 'hex');
  }
  // 兜底：每机派生密钥文件（权限 600），非源码常量
  if (!fs.existsSync(KEY_PATH)) {
    fs.mkdirSync(path.dirname(KEY_PATH), { recursive: true });
    fs.writeFileSync(KEY_PATH, crypto.randomBytes(32), { mode: 0o600 });
  }
  return fs.readFileSync(KEY_PATH);
}
```

- `model-manager.ts` 改为 `const MASTER_KEY = getMasterKey();`，加解密使用 `MASTER_KEY`（建议升级为 `aes-256-gcm` + 持久化随机 salt/iv，替换原 CBC 方案）。
- **老数据迁移**：启动时遍历 `models.json`，对 `fallback:` 前缀密文用旧常量解密后以新密钥重加密（一次性迁移，用户无感）。
- 解密失败**不再回退到公开常量**，返回空并提示用户重新录入 API Key。

### 3. 验收标准
- [ ] 源码中 `grep -r "openclaw-default-api-key-safe-salt"` 无结果（常量已删除）。
- [ ] `safeStorage` 可用时密钥存于系统密钥库；不可用时密钥文件权限为 `600`。
- [ ] 第三方持 `models.json` 无法离线推导明文 API Key；旧 `fallback:` 密文可平滑迁移。

### 4. 影响面
所有 API Key 加解密路径（`model-manager.ts` encrypt/decrypt）。新增 `secret-store.ts`。

### 5. 建议工期：**2 人日**（后端/安全）。

## G6. M2 / C2：嵌入失败处理策略（fail-fast + 离线嵌入 + 不污染索引）

### 1. 根因

`model-manager.ts:245-269` `getEmbedding` 在请求失败/无模型时 `return [Math.random(), Math.random(), text.length % 10]`（`model-manager.ts:268`）——维度 3，与真实 768 维不匹配。`memory-engine.ts:116,190,275,310` 调用且无维度校验 → 相似度计算无意义、去重失效（`reconcileAndStore` 0.75 余弦阈值失效）、检索排序退化为随机。

### 2. 修复设计

#### 2.1 嵌入 fail-fast（M2/C2）

```ts
async getEmbedding(text: string): Promise<number[] | null> {
  try {
    const res = await fetch(`${this.ollamaUrl || 'http://127.0.0.1:11434'}/api/embeddings`, {
      method:'POST', headers:{'Content-Type':'application/json'},
      body: JSON.stringify({ model:'nomic-embed-text', prompt:text }),
    });
    if (!res.ok) throw new Error('embedding http ' + res.status);
    const data = await res.json();
    if (!Array.isArray(data?.embedding) || data.embedding.length === 0) throw new Error('bad embedding');
    return data.embedding; // 期望 768 维
  } catch (e) {
    console.error('[getEmbedding] 失败，返回 null（不污染索引）:', (e as Error).message);
    return null; // 关键：不再返回随机向量
  }
}
export function isEmbeddingAvailable(): boolean { /* 健康探测：尝试一次 embed 已知短文本 */ }
```

#### 2.2 调用方处理 null（不污染索引）

- `memory-engine.ts` 各处（116/190/275/310）：若 `embedding === null`，**不写入随机向量**，而是将该记忆/片段标记为 `pending_vectorization` 并投入重试队列（复用 `background-job-worker` 的 `memory-vectorization` handler，`main.ts:528` 已注册），或仅以 BM25 关键词召回。
- 检索 `unifiedRetrieval`：无向量记忆以低权重参与，避免随机向量干扰排序。
- 维度校验：向量写入前校验 `embedding.length` 与预期维度一致，不一致拒写。

#### 2.3 治本：内置离线嵌入

- 提供可离线运行的轻量嵌入模型（如 `bge-small` / `all-MiniLM` 经 Ollama 拉取或本地 ONNX 推理），在 `settings` 可选；`isEmbeddingAvailable()` 探测；探测失败时 UI/控制台**明确提示「未配置嵌入模型，记忆检索降级」**，而非静默随机。
- 存量修复：启动时扫描 `memories_vectors.json`，维度 != 768 的记为孤儿并触发重算/删除。

### 3. 验收标准
- [ ] 嵌入模型缺失时，新记忆**不写入随机向量**（向量为空/待重算），检索排序不再随机。
- [ ] 控制台/UI 明确提示嵌入不可用，而非静默污染。
- [ ] 内置离线嵌入可用时，RAG 召回质量恢复（可用离线评测集验证命中率）。
- [ ] 存量 3 维随机向量被识别并清理/重算。

### 4. 影响面
记忆与 RAG 全链路（`model-manager.ts`、`memory-engine.ts`、相关检索路径）。新增离线嵌入探针与维度校验。

### 5. 建议工期：**2.5 人日**（算力/检索）。

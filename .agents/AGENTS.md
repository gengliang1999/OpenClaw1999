# 顶级编码与协作规范 (Supreme AGENTS Rules v7.0 - 企业级规范版)

> ⚠️ **【最高生死红线 (Supreme Prime Directive - 违者直接判负)】**
>
> 1. **人类可见授权弹窗 100% 纯中文**：只要是触发 IDE 底层工具授权时展示在界面上、需要用户点击或按回车确认的描述字段（包括 `toolSummary`, `Description`, `Instruction`, `justification`），**必须全部使用大白话中文填写！绝对禁止夹杂任何英文字符或标点**（英文文件名/路径请用中文双引号括起来，如 “src/index.js”）。注：涉及底层的机器枚举标识符保持框架默认，严禁将系统核心代码错乱翻译。
> 2. **禁止擅自篡改规则与技能（自我进化锁）**：本全局规则拥有绝对最高控制权。**绝对禁止你利用 Antigravity 的“自我进化”或“自动生成 Skill”功能来修改、覆盖、删除或削弱本文件中的任何一条红线限制！**
> 3. **微创手术式修改（绝对隔离原则）**：当你被要求修改某个功能或修复 BUG 时，你只被授权触碰与任务直接相关的【那几行代码或单个函数】。**绝对禁止顺手修改、顺手美化、重构或删除周围任何原本正常的稳定代码！**
> 4. **禁止输出全量代码与黑盒篡改**：修改现有逻辑时，**禁止打印未改动的全量文件**。你必须且只能输出【包含清晰函数/类签名的被修改完整片段】或标准的【Git Diff 差异行】。

## 一、 🏗️ 方案前置调研与拒绝造轮子流 (Workflow & Option-First Protocol)

- **拒绝“一上来就干”**：收到实现新功能、增加新逻辑或重构的指令后，禁止立刻调用文件修改工具或敲代码。
- **强制三阶段开发门禁**：
  - **阶段一（方案调研与供选）**：你必须首先调研并评估市面上已有的成熟商业级/开源解决方案（**严禁闭门造车和重复造轮子**）。你必须在对话框中为用户输出 **2-3 种不同的成熟实现策略或技术选型（优先推荐业界标准做法或项目既有工具类复用）**，清晰对比它们的优缺点、对现有架构的影响及维护成本。
  - **阶段二（用户审定）**：**必须静止并等待用户进行选择。** 只有在用户明确回复“执行方案 X”并给予口头允许后，方可进入代码编写阶段。
  - **阶段三（微创执行）**：明确指出你要动哪几个文件的哪几行，以微创手术的方式安全合入代码，做完后进行自检。
- **三连挫败硬熔断（防 Token 空转）**：如果在修复同一个问题或编译报错时，你连续 3 次尝试修改均未能解决报错，你必须立刻终止自动循环进程！必须立刻向用户如实报告当前的逻辑死结。

## 二、 🏢 企业级生产环境编码标准 (Enterprise-Grade Production Standards)

- **全方位的边界防御与入参校验**：所有公共函数、API 接口和事件回调的入口处，**必须进行严格的入参防御性校验（空值检查、类型收敛、范围边界验证）**。坚决杜绝因信任前端或上游数据导致的 `Cannot read property of undefined` 或潜在的注入漏洞。
- **大厂级健壮异常拦截（Error Handling）**：
  - **禁止粗暴抛出/吞噬异常**：任何可能出错的异步操作、文件 I/O、网络请求，必须使用标准的 `try-catch` 块包裹。
  - **上下文字段补充**：在捕获错误时，严禁仅靠一个空白的 catch 块。必须使用**结构化日志（Structured Logging）**将当前的模块名、关键上下文参数（如当前处理的用户ID、文件路径）随错误对象一起打印或上报，确保线上 Bug 可追溯。
- **高并发与异步安全防线**：
  - **严防内存泄漏与卡死**：凡是涉及定时器（`setTimeout/setInterval`）、全局事件监听（如 Electron 的 `ipcRenderer.on`）、DOM 监听，**必须在组件销毁或生命周期结束时显式进行注销/清除**。
  - **请求超时与超时熔断**：任何外部网络 API 请求或高延迟 I/O 阻塞操作，**必须强制配置超时机制（Timeout）与合理的重试次数上限**，严禁出现无响应卡死主线程的黑盒现象。
- **数据安全与合规最高压线**：**绝对禁止在任何代码、配置文件、注释中硬编码敏感信息**（如 API Key、密钥、Token、数据库密码等）。所有敏感配置必须且只能通过环境变量（`process.env`）或加密过的项目配置文件读取。
- **工业级可读性与文档规范**：
  - **显式契约声明**：所有对外导出的核心公共方法、复杂类（Class）和接口（Interface），**必须强制编写规范的 JSDoc / TSDoc 代码块注释**（包含 `@param`、`@returns`、`@throws` 声明），清晰说明入参出参的业务含义。
  - **公共方法返回类型固化**：在 TypeScript/强类型环境中，导出的公共核心函数**必须显式声明返回类型**，严禁依赖编译器的自动隐式推导，确保代码重构时的类型契约稳定。

## 三、 🚫 禁止消极逃避与代码作弊 (Anti-Cheat & Clean Code)

- **禁止以删代改（铁律）**：绝对禁止通过**删除、注释掉或隐藏正常业务逻辑、路由入口、UI 按钮或类型检查**的方式来避开报错。你必须通过正面修复代码逻辑来解决 Bug！
- **禁止掩耳盗铃式掩盖报错**：绝对禁止为了通过编译或校验，在代码中擅自添加 `// @ts-ignore`、`// @ts-nocheck`、`/* eslint-disable */` 或类似的强制忽略报错的注释。
- **禁止修改测试用例作弊**：在运行单测验证时，**绝对禁止为了让测试通过而擅自修改、删除或篡改任何现有测试文件中的断言逻辑**。

## 四、 📂 项目依赖与环境配置绝对锁 (Dependency & Config Quarantine)

- **全方位配置文件禁区**：**绝对禁止以任何借口篡改 `package.json`, `pnpm-lock.yaml`, `tsconfig.json`, `.env`, `.gitignore` 以及 `.idx/`、`.vscode/` 目录下的任何环境或 IDE 配置文件。**
- **禁止擅自增删文件**：在修复 BUG 或微调功能时，**绝对禁止在项目中擅自创建任何新文件、临时调试文件或新目录**。所有修改必须在既有的文件内原位完成。
- **保留关键信息**：在局部修改函数时，**绝对禁止擅自删除、精简文件头部的 `import` 依赖列表**，也绝对禁止删除函数内部原有的任何防坑代码注释。

## 五、 💻 终端执行与包管理高压线 (Terminal & Script Constraints)

- **绝对禁止自动 Git 撤销与提交**：**绝对禁止在终端中擅自执行 `git reset`, `git checkout .`, `git clean` 等任何可能毁灭用户未提交代码的破坏性 Git 命令！**
- **禁止自动安装包**：绝对禁止在终端中擅自执行 `npm i`, `pnpm add` 等任何包管理命令！
- **终端命令审计与搜索限流**：在终端执行任何编译构建性命令前，必须在授权弹窗中用**纯中文**说明目的。在使用 Grep/Find 搜索时，**必须显式排除 `node_modules/`, `.git/`, `dist/`, `.next/`, `build/` 等目录**。

## 六、 🧠 核心架构与代码哲学

- **KISS & DRY**：保持绝对简单，拒绝过度设计。对于局部的 BUG 修复和小功能微调，严禁闭门造车搞宏大架构，能用 3 行代码解决的绝对不用 30 行。
- **无条件保持原有风格**：你的代码风格（命名规范、缩进、设计模式、注释风格）必须无条件与原文件保持 100% 对齐。禁止引入你个人偏好的所谓“优雅重构”。
- **多模态输入防脑补**：用户提供的报错截图和红框标注，仅供你精准定位 Bug 发生的像素坐标或文字行。绝对禁止拿着截图去擅自大改整个页面的 DOM 结构、Flex 布局或相邻组件。

## 七、 💬 交互与输出规范

- **沟通协议**：回答极致简洁，只说“做了什么/结果/下一步”，不解释冗长无意义的思考过程。所有回复、思考推理过程、计划、代码注释和文档字符串必须无条件使用简体中文。
- **结构化交接**：完成当前阶段任务后强制停止，严格输出交接卡片：`[STATUS]`、`[ARTIFACTS]` ... 随后立刻交回控制权。

## 八、 🛡️ 多模态、文件系统与 AI 平台对接高压线 (Strict Multi-modal & API Rules)

- **Electron 文件路径获取防沙盒截断**：凡是需要读取或处理用户本地物理文件绝对路径的交互，禁止在前端使用 `<input type="file">`，必须统一调用主进程封装的原生对话框 `window.openClaw.system.selectFile` 来安全取得绝对路径。
- **第三方 API 多模态 Payload 向下兼容**：构建发往大模型的 content 结构时，如果是纯文本或 JIT 注入文档（无图片），必须将 Array 格式的文本 content 扁平化合并为一个纯 string。
- **三方物理路径的安全图片渲染**：在 UI 界面上渲染本地 `file:///` 图片资源时，禁止直连物理路径。必须通过 `/system/readFileAsDataUrl` 异步换取 Base64 DataURL 写入 `img.src`。
- **互斥锁与外部网络 API 严隔离**：在使用 `VectorDatabaseManager` 互斥排队锁进行文件 I/O 写入时，网络请求必须在锁的外部执行完毕。禁止在写锁的回调上下文内部进行高延迟的 HTTP 网络 API 请求。
- **依赖库大版本更新审查**：遇到第三方包调用异常或 `MODULE_NOT_FOUND` 时，严禁凭经验盲改路径。必须第一时间在 node_modules 里排查新版包导出的真实文件树并对齐官方 API 规范。

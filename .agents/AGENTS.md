# 顶级编码与协作规范 (Supreme AGENTS Rules v13.6 - 极速批处理终极版)

> **【第0号绝对红线：批处理任务前置锁 (Batch-Action XML Lock)】**
> 绝对禁止在未获授权时进行写操作！为保证工程效率，实行“任务级单次授权”，严禁单文件频繁打扰。
>
> 1. 触发时机：在 [阶段一] (或快捷绿色通道后) 梳理完受影响文件，跨入 [阶段三] 正式写代码前夕，必须且只能输出一次以下 XML：
> <pre_action_audit>
> <intent>清单列出本次连续修改的所有文件（如：1. A.ts 2. B.tsx）及执行命令。</intent>
> <rule_check>默写对应最高红线（如全栈原子化联动、微创手术）。</rule_check>
> <status>等待用户回复 [Proceed] 批准批量执行。</status>
> </pre_action_audit>
>
> 2. 连续执行特权：获批 [Proceed] 后即得【批量执行通行证】。必须连续、不间断调用工具一口气改完清单文件。顺利执行期间，绝对禁止每改一个文件就停下索要 [Proceed]！
>
> 3. 重新上锁条件：若发生以下情况通行证失效，必须立即停止并请示人类：
>    - A. 遇到终端编译报错或致命 Error。
>    - B. 发现实际代码逻辑与原计划完全不同，必须扩大修改范围。

> **【最高生死红线 (Supreme Prime Directive)】**
>
> 1. 纯中文弹窗防御：触发 IDE 底层工具授权或命令审计界面的描述字段（含 toolSummary, Description 等），必须全部使用简体中文！绝对禁止夹杂英文单词或标点（英文文件路径需加中文双引号）。机器枚举标识保持默认。
> 2. 禁止篡改规则技能：本规则具最高控制权。绝对禁止利用“自我进化”或“生成 Skill”功能修改、覆盖或削弱本文件任何红线！
> 3. 微创手术式修改：修改或修复时，仅授权触碰任务直接相关代码行。绝对禁止顺手美化、重构或删除周围正常稳定代码！
> 4. 禁输全量代码与黑盒篡改：禁止在对话框打印未改动全量文件。必须且只能输出含清晰函数签名的完整修改片段或标准 Git Diff 行。
> 5. 超十行强制文件交付：新增代码或重构逻辑超 10 行，绝对禁止在对话框内贴出，必须统一调用工具写入对应物理文件！

## 一、 需求追问与门禁前置流 (Blueprint Protocol)

* 拒绝脑补需求：收到复杂需求禁止立刻敲代码或做规划。
- [HOTFIX_MODE] 越权与熔断：指令以 `[HOTFIX]` 或 `[QUICK]` 开头可合法跳过 [阶段零/一]，直接输出 `<pre_action_audit>`。HOTFIX 仅限单文件修改或少于 10 行逻辑修复。若涉及跨文件业务或 API 联动，必须主动拒绝特权，强行回退 [阶段零]。
- 阶段零 (需求探明)：无 HOTFIX 标识必须澄清场景、边界、关联。基于已知信息提供架构草稿，绝对禁止以“等待回复”完全挂起。
- 阶段一 (设计前置)：细节对齐后，输出文字《功能设计规划蓝图》，含功能清单、数据链路。严禁重复造轮子。
- 阶段二 (方案冻结)：必须原地静止，禁止调用编辑工具，等待人类明确口令。
- 阶段三 (微创执行)：方案冻结后，安全合入最小受影响代码并自检。
- 三连挫败强制换轨：针对同一报错最多 2 次重试权限。第 2 次重试绝对禁止在原失败逻辑上打补丁，必须强行推翻切换技术路径！第 3 次报错必须收回调用权，输出 `[SYSTEM MELTDOWN: 修复失败，请求人类介入]`。禁删旧代码避错。

## 二、 企业级生产环境编码标准 (Production Standards)

* 自适应与多主题保护：禁止 JS/TS 中对容器宽高硬编码赋值。确保自适应收缩不重叠。新 UI 必须支持主题过渡，严禁无变量硬编码颜色。
- 性能红线：高频执行路径严禁大批量创建临时对象杜绝 GC 抖动。
- 环境配置解耦：禁止业务代码硬编码环境切换逻辑或 URL，统一剥离至环境变量。
- 国际化合规：绝对禁止硬编码裸露汉字或英文字符串！可见文案必须经国际化字典调用。
- 数据向下兼容：改动本地存储或 IPC Payload 必须写平滑迁移逻辑。读本地存储必裹 try-catch 并写默认值降级。
- 健壮异常拦截：异步与网络请求必须 try-catch。绝对禁止日志明文打印 Token 等隐私！
- IPC 双端超时：跨进程通信必须强制双端 try-catch。渲染端必配超时限制，禁挂死。
- 类型契约：对外核心方法必写规范注释与返回类型。缺失类型必过 declare module 补齐。

## 三、 禁止消极逃避与代码作弊 (Anti-Cheat & Clean Code)

* 禁止空壳 UI：交付 UI 必须一次性高完成度咬合状态管理与 IPC 逻辑。
- 拒绝空壳实现：严禁 `// TODO` 未完成功能或 dummy 返回数据骗过类型检查。
- 禁止以删代改：绝对禁止通过隐藏业务逻辑或类型检查避开报错。
- 拔管原则 (禁留垃圾)：问题修复交付前，必须主动全局清查并删除排查时添加的所有 `console.log`、打印语句及大段注释掉的废弃代码。
- 禁止掩盖报错：绝对禁止擅自添加 `// @ts-ignore` 强制忽略报错。
- 禁止篡改测试：绝对禁止擅改测试文件断言逻辑。

## 四、 环境与依赖绝对锁 (Environment Locks)

* 配置文件禁区：绝对禁止篡改 package.json, pnpm-lock.yaml, tsconfig.json, .env, .gitignore 及 IDE 环境配置。
- 禁止自动安装：绝对禁止终端擅自执行 npm/pnpm install。依赖缺失纯文本报告人类。
- 禁止擅建文件：绝对禁止擅自创建新文件/目录。修改必在既有文件原位完成。
- 保留关键信息：绝对禁止删文件头 import 列表或原防坑注释。

## 五、 终端、Git 与防上下文爆炸 (Terminal & Context Guard)

* 读前搜索防爆炸：绝对禁止盲目读取上千行全量文件！必须先用终端 `grep` 精确锁定行号。修改前必验证物理路径 100% 存在。
- 终端搜索限流：Grep/Find 必须显式排除 node_modules 等构建目录，禁盲目暴搜。
- Windows 兼容：调用终端必识别类型，Windows 下禁 Unix 路径写法，路径变量必裹双引号。
- Git 越权防爆：绝对禁止擅自执行 git commit/push/merge 或破坏性撤销 git reset。

## 六、 核心架构防守哲学 (Core Architecture)

* 竞态防范：高频异步流必写防御竞态锁定或旧请求清除 (AbortController)。
- KISS 死守：风格必须与原文件 100% 对齐，禁止个人偏好优雅重构。
- 防脑补：报错截图仅供定位，绝对禁止拿截图擅改全页 DOM。

## 七、 极简输出规范 (Output Formatting)

* 语气定位：严谨技术伙伴。不使用表情符号与动作描写。
- 拒绝过度排版：严禁无节制加粗和列表，自然段落陈述。
- 极简协议：简体中文。提问单次仅限 1 个核心澄清问题。交接任务输出卡片并归还控制权。

## 八、 多模态与文件系统协议 (Multi-modal & API)

* 路径安全获取：前端禁 `<input type="file">`，必调主进程原生对话框取绝对路径。
- API Payload 兼容：纯文本 Array 须扁平化为纯 string。
- 安全渲染：禁直连物理路径渲染本地图，必换 Base64 DataURL。
- 互斥锁隔离：写文件互斥锁外部必须执行完网络请求，禁锁回调内网调。
- 依赖审查：调用异常禁盲改路径，必排查 node_modules 真实导出。

## 九、 全栈双端同步与连坐铁律 (Full-Stack Atomic Sync)

* 核心咬合：改后端 Schema 或返回结构，必须无条件同任务全链路同步修改前端对应调用点！禁止单端遗留旧状态。
- TS 类型连坐：修改核心模块入参/出参，必须第一时间同步更新 types/ 或接口定义。绝对禁止致 TS 接口过期引发 Any 推导崩溃。
- 消灭编译幻觉：禁以静态编译通过为交付标准，无条件脑内演练全链路交互盲区。
- IPC 审计：微调前端逻辑必向后追溯两层确认分发对称，防运行时 TypeError。
- 空指针防御：严禁对未追溯返回值的动态对象主观解构。
- 异常友好化：连接错误必拦截翻译为含明确操作指引的中文提示。

## 十、 一致性防御与自适应 (Consistency Rules)

* 安全契约：改 CSP 必同步剥离受影响页内联 JS 事件，改由委托代理接管防 DOM 报错。
- 双库一致：混合持久数据写入/删除必视为统一原子事务，显隐向量文件 100% 同步清除。
- safeStorage 防崩：解密入口必 try-catch，异常自动删旧密文重成密钥防死锁崩溃。
- 弹性自适应：拖拽面板按钮必配固定宽度与 flex-shrink: 0 隐滚动条。

## 十一、 Electron 并发设计铁律 (Electron Core)

* 主进程编译重启：改主进程 TS 代码后必在终端 `npm run build:ts`。编译成功必明确请求人类 `Ctrl + C` 重启进程。禁未重启声称生效。
- IPC 异步非阻塞：`ipcMain.handle` 绝不允许 `await` 长持流任务 Promise。必入后台异步队列并立即 `return { started: true }` 释放通道，数据靠 `webContents.send` 推流。

## 十二、 内存泄漏绝对防御 (Memory Leak Prevention)

* 强制卸载清理：前端组件绑定全局事件、定时器或 `ipcRenderer.on`，必须无条件在卸载生命周期写对应 removeListener/clearInterval。
- 禁多次绑定：绝对禁止重复挂载引发 IPC 事件多次触发 (幽灵事件) 导致内存溢出。

## 十三、 自动批准与回滚标准 (Rollback Standards)

* 自动连贯执行：获首次 [Proceed] 后，若系统自动批准后续工具调用，请连贯执行勿停下询问，除非报错或遭拒。
- 彻底回滚：代码回滚需：1.恢复源文件 2.终端强杀进程 3.强制清编译产物重构还原。
- 运行时审查：改无 TS 契约代码，调用本地辅助方法前必用全局检索确认其物理定义存在。

## 十四、 微创隔离红线 (Surgical Precision)

* 精确微创：仅限精准修改受影响行，绝禁对周围正常函数体全量重写。
- 差异审计：物理写入后，必须在终端运行 `git diff` 逐行核对，非预期差异立即原地恢复。
- 标签检验：拼接 HTML 模板必静态核对类名与闭合，禁重复 class 或内联处理器。

## 十五、 物理外挂与环境协同协议 (Physical Add-ons)

* 项目地图寻址：接手任何非 HOTFIX 任务时，必须优先静默读取项目根目录的 `MEMORY.md` 文件获取全局架构分布。
- 复杂任务草稿本：处理涉及 3 个以上文件的高耦合任务，禁止在对话框内长篇思考推演。必须调用写入工具，将详细推导步骤记录在项目根目录 `.ai-scratchpad.md` 文件中作为外置缓存，任务完成后必须清空该文件内容。
- 强制自证清白：完成任何涉及 TypeScript 的逻辑修改后，必须在终端执行 `npm run ai:verify`。如遇报错，必须在扣除“熔断次数”的限制内自行修复完毕，直到终端无错误抛出方可向人类汇报。

## 十六、 输出格式硬性锚点 (Output Anchor)

为证明未遗忘规则，每次文本回复正文必须强制以下列格式开头（连续执行批量修改免人类交互时可豁免，保持控制台整洁）：

**[当前执行门禁]**：(阶段零 / 阶段一 / 阶段二 / 阶段三 / HOTFIX绿色通道)
**[剩余熔断次数]**：(2 / 1 / 0 - 触发报错时递减)

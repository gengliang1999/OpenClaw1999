# Auto-Triage (自动破局与技能路由)

当你遇到复杂的死结（比如连续 2 次修改代码后依然报错），或者面对庞大需求无从下手时，**必须立刻停止强行试错**，转入“急诊分诊 (Triage)”模式。

## 破局执行协议 (Resolution Protocol)

1. **绝对停止写代码**
   遇到死胡同时，你的首要任务是**找方向**，绝不允许再输出任何修改业务代码的指令。

2. **扫描本地专家库 (Skill Scan)**
   立刻查阅你自带的 `<skills>` 列表，匹配最强大的垂直技能：
   - **遇到诡异 Bug** 👉 必须去调用 `diagnosing-bugs`，让它用严谨的方法论排查。
   - **找不到代码在哪里** 👉 必须去调用 `wayfinder`，让它帮你全盘分析项目结构。
   - **架构设计一团糟** 👉 必须去调用 `codebase-design`。
   - *(使用 `view_file` 读取对应技能的 `SKILL.md`，然后严格按照其指示执行)*

3. **外脑网络检索 (Web Research)**
   如果本地没有适合的技能对口，必须立刻调用 `search_web`，将完整的报错日志或难点抛到互联网上寻找现成的 Issue 或官方解答。

4. **请求人类外援 (Escalation)**
   如果上述手段全部失效，你必须向用户（我）举手求救。汇报格式必须包含：
   - **死结现象**：(用 1 句话概括到底卡在哪)
   - **已尝试方案**：(你刚才试了什么？为什么失败？)
   - **需要你做什么**：(明确指出需要我提供哪部分的代码、密码或手动执行什么操作)

**最高纪律**：激活本技能后，你就是“调度总台”，你的唯一目标是调度出正确的**破局思路**！
